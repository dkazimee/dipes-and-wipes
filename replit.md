# Dipes & Wipes

A smart baby subscription service that automatically sizes up diapers as a baby grows. Parents enter their newborn's weight, height, and age percentiles and the app ships the right size diapers every month — automatically.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/dipes-and-wipes run dev` — run the frontend (port 24456)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite + Tailwind CSS + Framer Motion
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod, drizzle-zod
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)
- Fonts: Fraunces (serif), Outfit (sans)

## Where things live

- `lib/api-spec/openapi.yaml` — OpenAPI spec (source of truth for all API contracts)
- `lib/db/src/schema/` — Drizzle DB schema (babies, growth, products, subscriptions, orders)
- `artifacts/api-server/src/routes/` — Express route handlers
- `artifacts/dipes-and-wipes/src/pages/` — React pages
- `artifacts/dipes-and-wipes/src/components/` — Shared components

## Architecture decisions

- Diaper sizing is calculated server-side from baby's weight using CDC weight ranges (Preemie < 6lbs → Size 6 35+lbs)
- Growth entries auto-update the baby's current weight and diaper size when submitted
- Subscriptions store the current diaper size and next delivery date; size is synced from the baby profile
- Orders are pre-generated for upcoming deliveries with `pending` status
- All API contracts are OpenAPI-first; frontend hooks are generated by Orval

## Product

- **Landing page** — marketing page with hero, how it works, features, pricing
- **Baby profiles** — add babies with name, birthdate, weight/height percentiles
- **Growth tracking** — log weight/height from each pediatrician visit
- **Smart sizing** — automatic diaper size calculation based on weight ranges
- **Subscriptions** — monthly or bimonthly delivery subscriptions with product selection
- **Pause/resume** — pause or cancel subscriptions anytime
- **Product catalog** — browse diapers, wipes, creams, and essentials
- **Order history** — track upcoming and past deliveries

## Gotchas

- Use `zod` (not `zod/v4`) in api-server routes — esbuild can't resolve the `zod/v4` subpath
- Run codegen after every OpenAPI spec change: `pnpm --filter @workspace/api-spec run codegen`
- Font imports must be at the very top of index.css (before all other @import rules)
